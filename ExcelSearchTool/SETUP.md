# 🔧 상세 설정 가이드

## 개발 환경

| 항목 | 내용 |
|---|---|
| IDE | Visual Studio 2019 |
| 프레임워크 | .NET Framework 4.7.2 ~ 4.8 |
| 프로젝트 유형 | Windows Forms 앱(.NET) |

---

## 1. NuGet 패키지 설치

> **EPPlus → NPOI 결정 과정**
> 처음에는 EPPlus로 시작했지만, 검색 대상에 `.xls`(2003 이전 구버전) 파일이 포함될 수 있었습니다.
> EPPlus는 `.xlsx`만 지원하므로, `.xls`/`.xlsx` 둘 다 지원하고 라이선스 제약이 없는 NPOI로 전환했습니다.

패키지 관리자 콘솔에서:
```
Install-Package NPOI
```

---

## 2. Properties.Settings 설정 항목

> **설정 파일 분리 이유**
> 기본 폴더 경로, 이동 폴더 경로, 검색 옵션을 코드에 하드코딩하면 사용자가 직접 코드를 수정해야 합니다.
> `Properties.Settings`(사용자별 config, %AppData%에 저장)로 분리하여 프로그램 내 설정 화면에서 변경 가능하도록 했습니다.

프로젝트 우클릭 → 속성 → **설정(Settings)** 탭에서 아래 항목 추가:

| 이름 | 형식 | 범위 | 기본값 |
|---|---|---|---|
| `DefaultFolderPath` | string | User | (빈 값) |
| `MoveFolderPath` | string | User | (빈 값) |
| `CaseSensitive` | bool | User | False |
| `IncludeHiddenSheets` | bool | User | False |

> 설정 탭이 보이지 않으면 "이 프로젝트에는 기본 설정 파일이 없습니다. 만들려면 여기를 클릭하세요" 링크를 먼저 클릭합니다.

## 3. 작업 폴더 구성

> **폴더 구조 결정 과정**
> 검사 대상 폴더와 이동 대상 폴더를 분리할지, 같은 폴더의 하위 폴더로 둘지 고민했습니다.
> 이동 폴더를 완전히 별도 경로로 지정 가능하게 하여, 검사 폴더와 후속 처리 폴더를 자유롭게 분리할 수 있도록 했습니다.

```text
[기본 폴더] (설정 > 기본 폴더 경로)
  ├── A.xlsx
  ├── B.xls
  └── ...

[이동 폴더] (설정 > 이동 폴더 경로, 완전 별도 경로 가능)
  └── (검색어 포함 파일들이 여기로 이동됨)
```

---

## 4. 최초 실행 시 설정 순서

1. 프로그램 실행 (최초에는 폴더가 비어있는 상태)
2. "설정" 버튼 클릭 (메인 폼 우측에 설정 창이 열림)
3. **기본 폴더 경로**: "폴더 선택" → 검사할 엑셀 파일들이 있는 폴더 지정 → 저장
4. **이동 폴더 경로**: "폴더 선택" → 검색어 포함 파일을 모아둘 폴더 지정 → 저장
5. **대소문자 구분 / 숨김 시트 포함 여부**: 라디오버튼 선택 → 저장
6. 설정 창 닫으면 메인 화면에 기본 폴더의 파일 목록이 자동으로 표시됨

---

## 5. 설정 초기화

설정 화면의 "초기화" 버튼 클릭 시:
- 기본 폴더 경로 / 이동 폴더 경로 → "(지정안됨)"
- 대소문자 구분 → 구분안함
- 숨김 시트 포함 → 포함안함

으로 모두 되돌아갑니다. 메인 화면도 자동으로 갱신되어 폴더/파일 목록이 비워집니다.
